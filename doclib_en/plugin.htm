Introduction

In HSP, it is possible to add functions in the form of extension plugins and modules. This makes it possible to perform processes that cannot be achieved with the HSP core alone.

HSP extension plugins are standard files with the DLL or HPI extension. This file must exist in the same directory as the HSP core (HSP3.EXE). Also, it cannot be placed in PACKFILE when creating an EXE file to make it a single EXE file. When you create an EXE file that uses an extension plugin, place the extension plugin in the same directory as the EXE file.

Modules are files for adding functions like extension plugins, but they are used by adding them to the source script. Unlike HSP extension plugins, modules do not need to be placed in the same directory as the EXE file.

The following extension plugins are included as standard. Please refer to the separate text for details on how to use each plugin.

Extension Plugins

HSPEXT.DLL

Introduction of HSPEXT.DLL
Extension Plugin HSPEXT.DLL --- Author: onitama Author's Page

This is a function extension plugin that widely supports functions other than the standard functions. By using this extension DLL, more than 40 new commands will be added, and many functions such as registry operations, simple mathematical functions, serial communication, extended image operations, and extended file access can be controlled from HSP.

Furthermore, by installing an application capture command, it becomes possible to control applications other than HSP.

These commands are useful for creating tools and utilities that cannot be achieved with HSP alone, and enable even more advanced application development.

For details, please refer to the separate document "hspext.txt".

HSPINET.DLL

Introduction of HSPINET.DLL
Extension Plugin HSPINET.DLL --- Author: onitama Author's Page

This is a plugin for internet access. It allows you to easily access http and ftp servers.

In addition to acquiring data on a specified page, a wide range of options are available, such as accessing via a proxy server and acquiring update dates and times. When connecting via ftp, detailed operations are possible, from directory listing to sending ftp commands. In addition, commands for acquiring the CRC42 and MD5 values of files are also included.

For details, please refer to the separate document "hspinet.txt".

HGIMG3.hrt

Introduction of HGIMG3.hrt
Extension Runtime HGIMG3.hrt --- Author: onitama Author's Page

This is a runtime with high-performance 2D and 3D image processing functions. It is a DirectX-only function that supports 2D and 3D graphic display.

It can be used for a wide range of applications, from 2D games using sprites to full-fledged software using 3D models. The main functions added by HSP3HG.hrt are as follows.

  • Advanced drawing system for DirectX and full-color mode
  • Drawing engine that allows 2D sprites and 3D polygons to be set with common commands
  • Rotation, enlargement/reduction, alpha blending, transparent color removal, and tiling (BG display) of 2D sprites
  • World management that allows 2D sprites and 3D polygons to be mixed on 3D coordinates
  • Allows management of polygon models with 3D hierarchical structure
  • Allows animation display of 3D one-skin models using bones
  • Allows animation-style rendering (toon shading) display
  • Texture support for BMP, JPG, GIF, PNG, DDS, TGA, and TIFF formats
  • Texture rendering function for Japanese fonts including anti-aliasing
  • Support for texture images including alpha channels
  • Sprite display, movement management, and collision detection functions
  • Geometry calculation using floating-point numbers compatible with HSP3
  • Event list registration function enables highly flexible autonomous processing
  • Equipped with easy-to-use direct drawing commands
  • Equipped with sound functions including stream playback

It is not included with the main body from HSP3.7β8 onwards. You can download it separately using the HSP3 Updater (HSP3Update). Development of this runtime has now ended. Please use HGIMG4, which is its successor.

hsp3dish.hrt

Introduction of HSP3Dish
Extension Runtime hsp3dish.hrt --- Author: onitama Author's Page

This is a runtime with high-speed 2D drawing and sound functions. It can be run not only on Windows, but also on android, iOS, Linux, Raspberry Pi, and html5 (WebGL) without modifying the source code.

It includes standard sprites compatible with the HSPDX plugin, the OBAQ plugin, and simple network communication functions, and can be used for a wide range of applications from 2D games to full-fledged software. In addition, the HGIMG4 runtime, which has 3D functions, includes all the functions of HSP3Dish as a subset.

For more details, please refer to the separate document "hsp3dish_prog.htm".

HGIMG4.hrt

Introduction to HGIMG4.hrt
Extension Runtime HGIMG4.hrt --- Author: onitama Author's Page

This is a runtime with high-performance 2D/3D image processing functions. It provides OpenGL/DirectX9 functionality supporting advanced 2D/3D graphics display.

It can be widely used, from 2D games using sprites to full-fledged software using 3D models. The main features added by HGIMG4 are as follows:

  • Drawing with OpenGL/ES or DirectX9, shader language support
  • Scene management and integrated physics engine (Bullet) support
  • FBX format model file loading, animation support
  • Support for multi-texture, bump map, height map, and LOD
  • Font data (TTF) rendering support
  • Off-screen rendering and various post-effect support
  • Supports operation on Windows as well as iOS, Android, Linux, html5 (WebGL)
  • License terms and open source similar to HSP and HSP3Dish

For more details, please refer to the separate document "hgimg4.html".

OBAQ.DLL

Introduction to OBAQ for HSP3
Extension Plugin OBAQ for HSP3 --- Author: Zener Works, Inc./kuni/onitama Author's Page

This plugin allows you to easily utilize the functions of the physics engine (OBAQ). By using OBAQ, you can handle 2D-based rigid body collisions and gravity simulation. OBAQ specializes in 2D and has simple settings and commands so that anyone can easily realize natural movement through physical calculations. Please consider it as an easy-to-use system for supporting games and demonstrations, rather than a highly accurate technical calculation.

OBAQ has a built-in sprite manager, and you can easily perform sprite drawing using only standard functions. Also, by managing the display coordinates by the user, it is possible to support display systems using DirectX, etc.

For more details, please refer to the separate document "obaq.txt".

HSPCV.DLL

Introduction to HSPCV.DLL
Extension Plugin HSPCV.DLL --- Author: onitama Author's Page

This plugin allows you to easily utilize the functions of OpenCV.

By using HSPCV, you can handle advanced image processing and algorithms. In addition, it provides many functions related to video, such as reading and writing various formats (PNG, JPEG, TIFF, etc.), as well as movie (avi) file operations and reading capture devices such as USB cameras. The main features added by HSPCV.DLL are as follows:

  • Arbitrary size image image operations (copy, addition, subtraction, multiplication, etc.)
  • High-quality resizing, smoothing, and rotation of image images
  • Extraction, inversion, and color conversion by threshold from image image
  • Reading and saving image files compatible with various formats
  • Image acquisition via capture devices such as USB cameras
  • Reading and saving avi video files
  • Image recognition such as face extraction
  • Template matching to extract regions close to an arbitrary image

For more details, please refer to the separate document "hspcv.txt".

Artlet2D

Introduction of Artlet2D
Extension Module Artlet2D Ver1.01 R3 Author: S.Programs Author's Page

Artlet2D is a module for handling image data with alpha channels (transparency components) in HSP using GDI+. In addition to HSP's standard drawing commands, the following functions become available.

  • Virtual image buffer with pixel format 32bit ARGB
  • Mutual image copy between HSP screen and Artlet2D virtual image
  • Drawing functions for basic figures such as line segments, ellipses, and rectangles, and free curves
  • Drawing with anti-aliased smooth edges
  • Color command with alpha value
  • Gradient brush, texture brush
  • Free pen width, various pen styles
  • Text display that can be automatically wrapped and horizontally and vertically aligned within a rectangular area
  • Smooth image scaling and copying
  • Transparency of specified color range, gamma adjustment, brightness adjustment using color matrix,
  • Copy modes such as semi-transparent and sepia conversion
  • Changing the drawing position and rotation angle using a coordinate transformation matrix
  • Reading and writing png files with alpha channels

    • This module was created by S.Programs and is included with the HSP main unit. Please direct any opinions or feedback regarding the module to S.Programs, not onion software.

    For details, please refer to the separate document "Artlet2D.txt".

    SQLele

    Introduction of SQLele
    Extension Module SQLele Ver1.16 Author: S.Programs Author's Page

    SQLele (スクレレ) is a module that makes it easy to handle SQLite in HSP.
    For example, file lists for media players, address book and calendar data, Undo/Redo management for image processing software, character dialogue in games, save data, and high scores...
    Most users who have used HSP to some extent have had the experience of designing data structures and operating them with string manipulation commands and memory manipulation commands to manage such internal application data, or writing routines for sorting and extraction.
    For many apps, this is sufficient in terms of functionality and performance, but if you want to efficiently develop apps that handle complex data, why not try SQLite, a free SQL database engine?
    With SQLite, you can use an SQL database within your HSP app simply by adding a single DLL. You don't need to install a DB server or configure ODBC settings.

    This module is included with the HSP main unit, produced by S.Programs. Please direct your opinions and comments about the module to S.Programs, not onion software.

    For details, please refer to the separate document "SQLele.txt".
