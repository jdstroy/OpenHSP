What is HSP?

HSP is a scripting language system that anyone can easily use. You can easily build image-based game programs, screen savers, tools, and practical software applications simply by writing scripts.

HSP has the following features:

  • You can start using it immediately with a free development environment.
  • Beginners can handle it with an easy-to-learn instruction set.
  • Programs you create can be converted into executable files and freely redistributed, with no license fees required.
  • Numerous functions such as 3D display and communication can be added via extension plugins and modules.
  • It has a large user base and a track record of over 30 years.
  • Equipped with detailed manuals and a reference help system.
  • As a purely domestic tool, there are also abundant book resources.
  • Software created with HSP3Dish conversion function can be run on android, iOS, Linux, WEB browsers (html5).
  • Libraries for hardware control such as USB-IO and Arduino are included.
  • The interpreter operation specification is published as open source.

HSP can be used in various situations depending on your ideas.

  • From visual novels to full-scale 3D shooting games, you can create games in various genres.
  • Create desktop accessories and screen savers.
  • For impromptu tools such as screen layout prototypes and image data conversion.
  • For media browsers such as program launchers, menus, CG collections, and music collections.
  • In addition, full-scale applications can also be developed.

Operating Environment

HSP3 can operate the development environment on Microsoft Windows, Linux/RaspberryPi. Refer to the manual for each platform for detailed installation and usage instructions.

HSP3 Windows Version Manual
HSP3 Linux/RaspberryPi Version Manual

Refer to the Programming Manual for HSP3 syntax and programming rules.

The Windows version is the only development environment that supports the full functionality of HSP3. The Linux/RaspberryPi version only supports the command line version and HSP3Dish.
Source code created in these development environments can be run on android, iOS, html5 (WebGL). (There are some restrictions for each platform) Refer to the HSP3Dish Programming Manual for details on multi-platform development.

Future Plans

HSP3 development will continue in the future, aiming for an environment where ideas can be easily put into shape in a short amount of time.
We would like to incorporate what we can implement from the many requests and bug reports we have received. Future plans and future considerations are as follows.

HSP3.7

We will continue to update the update version for functions that could not be implemented in HSP3.6. The functions that are being considered for future implementation are as follows. (This does not mean that they will be performed in this order.)

  • Renewal of packfile, review of encryption function
  • Correction of HSP script editor and expansion of input/debug support
  • Organization and expansion of module functions
  • 32-bit support and speed-up of HSP drawing buffer
  • Sharing of code and materials using network connection, support for community
    • HSP4

    HSP2 appeared in 1997, and HSP3 appeared 8 years later, but we hope to connect it to the next generation along with changes in the times. The situation required for programming is also changing. While inheriting the ease of HSP, we would like to cut compatibility to some extent and create something that is easy to use, and announce it as a roadmap in the future.
    Currently under consideration are complete UTF-8, 64-bit support, automatic code analysis and learning functions, VM renewal (llvm, etc.), and native compilation (JIT).

    About the Management of Language Specifications

    The language specifications of Hot Soup Processor, including HSP2 and HSP3, are maintained and managed by Onion Software/onitama.

    Requests for changes or modifications to the HSP language specifications are accepted and considered on developer communities, bulletin boards, HSP3 WishList, HSP Wiki, etc.

    The processing system that is responsible for the core of HSP, such as interpreter code generation and execution, is published as open source as OpenHSP. By publishing the source code for HSP3 with an open specification, we aim to further improve the correspondence to many environments, reliability, and maintainability.

    Message from the Author

    HSP is not particularly compatible with other languages (although it is mostly BASIC), nor does it have a new language system.

    What HSP aimed for is an old-style computer system.

    Old computers were far less powerful in terms of memory and CPU speed than modern computers, but you could turn them on and use them right away, and if you typed RUN in BASIC, you could see the results right away. I feel that such an easy and casual system is being lost in modern computers.

    There are no advanced language specifications such as object-oriented programming here, but there is the ease of writing in the order you think of. And I think that's another way to go for small-scale development and programming learning for beginners.

    HSP is by no means a versatile system. If you want to do more advanced things or large-scale development with an interpreter, languages such as C#, Python, and Java would be more suitable. I hope that HSP can be nimble for small processes that don't require such advanced languages.

    (Onitama)
